复试项目课·Binoculars++ ·第1 课

大家肯定都遇到过这种情况：同学交了一篇论文，读起来特别“规范”——“首先⋯⋯其次⋯⋯此

外⋯⋯综上所述⋯⋯”——行文流畅、逻辑清晰，你凭直觉就觉得这是ChatGPT 写的，但你怎么科学

地证明它是AI 写的？

这就是AIGC（AI-Generated Content）检测问题：给你一段文本，判断它是人类写的还是AI 生

成的。

注意区分：这不是抄袭检测。知网查重查的是你的文字有没有和别人的重复——对AI 生成的内

容完全无能为力，因为AI 每次生成全新的句子，重复率为零。AIGC 检测是一个全新的问题：内容是

原创的，但作者不是人。

复试项目课·Binoculars++ ·第1 课

AIGC 检测方法分成两大流派：

流派1：训练式检测——用人类文本和AI 文本训练一个二分类器。代表方法：Ghostbuster（用

多个语言模型提取特征+ 线性分类器）、RoBERTa 分类器（微调预训练模型）、GPTZero（商业产

品）。致命缺陷：只对训练过的AI 模型有效。你用GPT-3.5 的数据训出来的检测器，检测GPT-4 就

不灵了。论文原话：“trained detection models often fail to transfer to new models”。而且标注成本极

高——大模型迭代速度极快（2023-2025 年出了几十个模型），每出一个新模型就得重新收数据、重新

标注、重新训练，根本跟不上。

流派2：统计签名检测——利用AI 文本的统计特征做检测，不需要训练数据。代表方法：Detect-

GPT（对文本做小扰动，看困惑度变化）、Binoculars（我们的baseline，用两个模型的困惑度比值）。

优点是零样本（zero-shot）——能检测任意AI 模型。

Base 模型vs Instruct 模型

Base 模型只会“接话”——你给它开头，它往后续写。你说“中国的首都是”，它续写“北京，位于

华北平原，人口约2100 万⋯⋯” 滔滔不绝。它不知道什么是“问题”。例：Falcon-7B、LLaMA-2-7B。

Instruct 模型会“听指令”——你问它问题，它回答问题。你说“中国的首都是什么？” 它简洁回答

“北京。”ChatGPT 就是Instruct 模型。例：Falcon-7B-Instruct。

关系：Instruct = 在Base 上用（问题, 回答）数据做微调。它们共享tokenizer 和90%+ 的参数。

这就是Binoculars 选它们做“双望远镜” 的原因——足够接近但有差异。

## 自回归生成

## LLM 生成文本的方式叫自回归生成——一次只生成一个token。例子：生成“今天天气真好” 时，

模型的工作流程是：给定“今天”→预测“天气”(0.3)；给定“今天天气”→预测“真”(0.2)；给定“今天

## 天气真”→预测“好”(0.8)。每一步都在问同一个问题：给定前面的字，下一个字概率最高的是什么？

数学表达：P(xt|x1, x2, . . . , xt−1)。关键性质：AI 生成的文本，每个token 都是模型认为“最合适

def __init__(self, observer="falcon-7b",
performer="falcon-7b-instruct"):
self.observer = AutoModelForCausalLM.from_pretrained(observer)
self.performer = AutoModelForCausalLM.from_pretrained(performer)
self.tokenizer = AutoTokenizer.from_pretrained(observer)

def compute_score(self, text):
encodings = self.tokenizer(text) # Step 1: tokenize
obs_logits = self.observer(**encodings).logits # Step 2
perf_logits = self.performer(**encodings).logits # Step 3
ppl = perplexity(encodings, perf_logits) # Step 4: log PPL
x_ppl = entropy(obs_logits, perf_logits, # Step 5: Cross-PPL

encodings, pad_token_id)
return ppl / x_ppl # Step 6: Binoculars Score!

def predict(self, text):
score = self.compute_score(text)

return "AI" if score < 0.9015 else "Human"

## 改进1 的技术细节——面试可能追问

Q：为什么用CV 不用标准差？
A：标准差受均值影响。一篇B=0.7 的AI 文本和一篇B=1.1 的人类文本，
绝对波动程度不同但相对波动才是我们要的信号。CV=std/mean 是无量纲的。
Q：句子太短会不会导致B 分数不准？
A：会。所以代码里有len(sent) > 10 的过滤。太短的句子（如“好的。”“嗯。”）
token 太少，PPL 估计不稳定。我们要求至少10 个字符（约5-8 个token）。
Q：一篇文章只有2-3 个句子怎么办？
A：句子数太少（< 3）时CV 不可靠。这种情况下我们退化为只用全文级B 分数，
多粒度特征权重自动降低。这是一个优雅降级的设计。
Q：段落级和句子级分析有什么区别？
A：粒度不同。段落级捕捉的是“主题切换” 的风格变化（人类不同段讲不同角度），
句子级捕捉的是“表达方式” 的变化（人类同一段内也会混合口语和书面语）。
两者互补，实测中段落级CV 的判别力比句子级稍强（0.22 vs 0.17 for human）。

## 实验方法论——5 折交叉验证

为什么不是简单的train/test split？
我们使用分层5 折交叉验证（Stratified 5-Fold CV）：
步骤：把1824 个样本分成5 份→每次用4 份训练GBM、1 份测试
→轮换5 次→每个样本都被测试过一次→汇总结果
分层：确保每一折中人类:AI 的比例和整体一致（约50:50）
优势：
• 所有数据都参与了评估→结果更稳定
• 避免train/test 随机划分的偶然性
• 对小数据集（1824 样本）特别重要
分类器选择：GBM（Gradient Boosting Machine）
34 个特征+1824 个样本→中小规模结构化数据
GBM 在这种场景下通常优于神经网络（更好的正则化+ 特征选择）

原因1：中文AI 的“套路” 更明显

Q1: 为什么不直接训练一个分类器，而是用Binoculars 这种零样本方法？
A1: 训练分类器泛化性差——在GPT-3.5 上训的模型，对GPT-4/Claude 效果可能很差。
Binoculars 零样本方法不依赖训练数据，天然泛化到新模型。

Q2: 变异系数（CV）这个指标为什么有效？有理论支撑吗？
A2: 有的。AI 用autoregressive 生成，每步选概率最高的token，
导致全文风格统一。人类写作受情绪、思路跳跃影响，
不同段落的“可预测性” 天然不同。CV 直接度量这种不均匀性。

Q3: 你说AI 转折词多，那如果人类也用很多“首先其次” 呢？
A3: 确实存在这种情况（比如百科文本）。所以我们不是只看转折词，
而是用34 个特征的组合判断。单个特征可能误判，
但多个特征互相印证时准确率就很高。

Q4: AUROC 0.949 是好还是不好？
A4: 仅用统计特征（不用LLM 推理）就达到0.949 是很好的。
如果结合Binoculars 的LLM 分数，预期能进一步提升到0.97+。

这道题90% 会被问到，要准备一个深入的回答：

Q: 既然你用了GBM 分类器，那不就是训练式方法吗？跟流派1 有什么区别？
A: 区别非常大，在两个关键点上：
第一，我们训练的不是“检测某个特定AI 模型”，而是“识别AI 的通用写作模式”
流派1 的分类器学到的是“GPT-3.5 爱说Additionally” 这种模型专属特征
我们的GBM 学到的是“AI 文本句长CV 低、转折词多” 这种跨模型通用特征
第二，标注成本完全不同
流派1 需要对每个新AI 模型收集数据+ 标注+ 重训——大模型迭代极快，根本跟不上
我们的34 个特征是纯统计计算，不依赖于知道文本是哪个模型生成的
HC3 数据集只是用来验证效果，不是用来“训练检测器”
第三，Binoculars 的LLM 分数部分是完全零样本的
PPL/Cross-PPL 比值不需要任何训练数据。统计特征部分虽然用了分类器，
但特征本身反映的是自回归生成的本质，而非某个模型的“方言”

但这样做会降低生成质量——这是一个trade-off。
务实的立场：在当前和可预见的未来，检测是有效的。
关键是做好多信号融合——单一方法可能被规避，
