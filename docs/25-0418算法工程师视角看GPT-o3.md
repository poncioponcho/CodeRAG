## 算法工程师视角看GPT-o3​

## 一、视觉推理的范式突破：从静态分析到动态交互

1.1 福尔摩斯GPT-o3​

OpenAI的GPT-o3在视觉推理领域展现了令人惊叹的能力。o3采取了多模态思维链（CoT）方法。这种

方法不仅允许模型主动进行图像剪裁和变换，还能借助外部工具如Python分析数据，进一步丰富其推

理过程。通过大规模的强化学习，模型逐渐学会了如何审时度势，试错调整策略，从而显著提升其解

决复杂数学、编程、科学等问题的能力。

一个有趣的case：​

## o3的工具调用机制与传统模型不同，它允许模型在思维链中嵌套工具使用。例如，在分析复杂电路图

## 时，模型可能首先调用图像剪裁工具聚焦关键元件，然后通过Python解释器运行电路仿真代码，最后

## 结合搜索到的行业标准进行验证。这种“视觉-代码-文本”的跨模态协同显著提升了推理的准确性和效

率。

## o3通过大规模强化学习训练，将人类反馈（RLHF）与环境交互结合。例如，在处理图像标注任务时，

模型会根据用户反馈调整剪裁区域的选择策略，逐步优化视觉推理的准确性。这种机制使得o3在数

学、编程等领域的错误率比前代模型降低20%。​

二、从隐式到显式：视觉推理的双重路径

2.1 智谱CogCoM的技术创新​

在GPT-o3展现动态推理能力的同时，智谱的CogCoM模型通过“操纵链”（Chain of

Manipulations）机制，开辟了视觉推理的另一条技术路径。例如，在分析卫星云图时，CogCoM会生

成一系列显式操作步骤：“找到东亚区域→裁剪并放大台风眼→识别云系结构→计算风速”，每个步

骤对应具体的图像处理函数（如`Grounding()`、`CropZoomIn()`），并生成可解释的文本描述。这

种方法将视觉推理过程分解为可追溯的操作序列，解决了传统模型黑箱化的问题。

数据生产流程

数据生产

该工作提出了一套高效的推理链数据生产框架，该框架包含三个主要阶段：

1. 在第一阶段，针对每一个视觉问题 Q，该研究基于语言标注者让其选择性地利用操纵来撰写推理步

骤，其中操纵所请求的结果以变量代替；

2. 在第二阶段，该研究基于视觉标注者来补充第一阶段中操纵所请求的视觉内容。视觉标注将使得线

性推理步骤变成由同一请求所返回的多个视觉内容来分叉的数

3. 在第三阶段，该研究对已经完成语言和视觉标注的树进行遍历，获得那些叶子节点与答案 A 相同的

路径，作为最终有效推理链。

通过采用 GPT4 作为语言标注者，GroundingDINO 和 PaddleOCR 作为视觉标注者，该框架可以高

## 效、自动地构建大规模的推理链数据。同时，同样的方式以人类作为标注者也为构建更加复杂的推理

链提供可能。

## 另外，由于图像缩放是人类行为，该研究在最终操纵链中满足一定条件（如 box 小于原图的 1/6）的

操纵之前随机插入缩放操纵（CropZoomIn）以补充。基于这种方式，该工作构建了 70k 条推理链数

据，并对数据进行了全面的分析。

模型架构

CogVLM 是一个通用的大规模视觉语言模型框架，其通过引入视觉专家来对不同的模态进行深度融

合。本工作的模型框架基于 CogVLM，具体来讲，模型包含四部分：1）Vicuna-7B-1.5 的语言基座；

2）EVA2-CLIP-E 的视觉编码器；3）基于 SwiGLU 的映射层；4）视觉专家模块。模型总体包含 17B 参

数，其中视觉专家模块占比 6.5B。​

在此基础之上，该研究设计了基于 KV-memory 的多图多轮视觉语言模型架构。即，在训练阶段，在

包含多次图片输入的一个推理链轮次，其保存 Transformer 每一层的 KV-memory 的累积，并在每一

步的 attention 计算时考虑当前累积的 KV-memory。​

当有新图片输入时，模型无需输入历史文本，而基于固定的 prompt 来要求模型基于历史 memory 进

行推理。这种方式使得模型能够在兼容已有 VLMs 架构的同时接受多次图片输入，并有效缓解了上下文

序列过长带来的问题。

CogCoM 基于两个主要阶段的训练，以构建具备推理能力的同时能够解决多个多模态任务的通用视觉

语言模型。

在第一阶段，其首先基于由 LAION-2B 和 COYO-700M 获得的 1.5B 图文对模型进行预训练，其次基

于由 LAION-115M 获得的 40M 清洗数据对模型进行 grounding 训练。​

在第二阶段，该研究通过混合涵盖四种能力（instruction-following、OCR、detailed-

captioning、CoM）的 570K 数据对模型进行对齐训练。​

虽然o3和CogCoM采用不同的技术路径，但它们共同解决了视觉推理的核心挑战：如何将图像信息转

化为可执行的推理步骤。o3通过强化学习实现工具调用的自主性，而CogCoM通过显式操作链提升可

解释性。这种差异并非对立，而是互补——例如，在医疗影像分析中，CogCoM的结构化操作链可用于

生成诊断报告，而o3的动态工具调用可用于跨模态数据整合（如结合患者病历和基因数据）。​

智谱的技术生态进一步强化了这种互补性。例如，GLM-4-Air的轻量化架构支持消费级显卡运行，而

CogCoM的操纵链数据生成框架已被用于医疗影像分析，研究者通过微调模型实现了对病理切片的自动

化标注。这种开源生态不仅加速了技术迭代，还促进了多模态推理的跨领域应用。

三、未来趋势：从单模态到全模态智能

4.1 技术挑战​

- 模态对齐：如何更有效地融合视觉、语言、音频等多模态信息，避免“模态鸿沟”问题。​

- 实时推理：在保证准确性的前提下，提升模型处理高分辨率图像和长视频的速度。​

- 伦理安全：防止模型生成误导性或有害的视觉推理结果，特别是在医疗、金融等关键领域。​

4.2 智谱的技术布局​

智谱的技术路线图显示，其下一代模型将聚焦于全模态智能：

- 多模态强化学习：通过模拟人类“观察-思考-行动”的闭环，训练模型在真实环境中执行复杂任务

（如自动驾驶、机器人操作）。

- 神经符号整合：将符号逻辑（如数学定理、物理定律）嵌入神经网络，提升推理的可解释性和可靠

性。

- 边缘端部署：优化模型架构，使其能够在手机、嵌入式设备等资源受限环境中运行。​
