## ✨ 新功能详解

### 1️⃣ HybridRetriever 混合分词算法 (hybrid_tokenize)

#### 背景问题

原有 `jieba.cut` 分词器在处理中英文混合文本时存在结构性缺陷：

| 缺陷类型 | 示例 | 影响 |
|----------|------|------|
| 公式拆分破坏 | `"d*d"` → `['d', '*', 'd']` | 数学公式检索失败 |
| 术语碎片化 | `"ResNet-50"` → `['ResNet', '-', '50']` | 技术术语匹配失效 |
| 复杂度表达式断裂 | `"O(n^2)"` → 拆分为 6 个字符 | 算法复杂度无法识别 |

#### 解决方案：双通道分词策略

```
输入: "d*d公式表示维度乘积，ResNet-50是经典CNN模型"
                    ↓
        ┌───────────┴───────────┐
        ↓                       ↓
   非中文序列提取            中文片段识别
   ["d*d", "ResNet-50",    ["公式表示维度",
    "CNN"]                  "乘积"，"是经典",
                           "模型"]
        ↓                       ↓
   完整保留为 token       jieba.cut 精确分词
        ↓                       ↓
        └───────────┬───────────┘
                    ↓
           合并结果（保持原始顺序）
```

#### 技术实现

**核心函数**: [`hybrid_tokenize()`](retrieval_core.py#L38-L75)

```python

    parts = re.split(pattern, text)
    
    # 分类处理并合并结果
    ...
```

**支持的文本类型**：

| 类别 | 示例 | 支持状态 |
|------|------|----------|
| 数学公式 | `d*d`, `O(n^2)`, `A*B`, `x_i` | ✅ 完整保留 |
| 技术术语 | `ResNet-50`, `BERT`, `GPT`, `CNN` | ✅ 完整保留 |
| 代码标识符 | `func(x)`, `ReLU`, `GPU`, `PyTorch` | ✅ 完整保留 |

### 2️⃣ 文档切分策略优化

#### 参数调整

| 配置项 | v2.3.x (旧值) | **v2.4 (新值)** | 变化幅度 |
|--------|--------------|----------------|----------|
| **max_chunk_size** | 1500 | **2000** | ⬆️ +33% |
| **chunk_overlap** | 0 (无) | **250** | ➕ **新增** |
| **h1/h2 边界处理** | 与其他级别同等对待 | **强制切分（最高优先级）** | 🔄 **逻辑重构** |

#### 核心改进

##### 改进一：h1/h2 标题边界优先切分

```python
# h1/h2 标题边界：始终强制切分（高优先级）
if level <= 2:
    flush_chunk()  # 立即在标题处分割
    current_sections = current_sections[:level-1]
    current_sections.append(line)
    continue
```

**设计原理**：
- 一级和二级标题代表文档的主要章节结构
- 在这些边界处切分可以保证章节的语义完整性
- 避免将完整的章节拆分到多个 chunk 中

##### 改进二：Chunk Overlap 机制

```python
# 应用 chunk_overlap：为相邻 chunks 添加重叠内容
if chunk_overlap > 0 and len(chunks) > 1:
    overlapped_chunks = []
    for i, chunk in enumerate(chunks):
        text = chunk.page_content
        if i > 0 and len(chunks[i-1].page_content) > chunk_overlap:
            prev_tail = chunks[i-1].page_content[-chunk_overlap:]
            text = prev_tail + "\n" + text
        
        overlapped_chunks.append(Document(
            page_content=text,
            metadata=chunk.metadata
        ))
    return overlapped_chunks
```

**设计原理**：
- 相邻 chunk 共享 250 字符的重叠区域
- 确保跨边界的知识点不会因切分而丢失
- 提高检索时的上下文连贯性

##### 改进三：降级切分策略

```python
# 内容长度检查：仅对非 h1/h2 边界的内容进行降级切分
content_length = len("\n".join(current_content))
if content_length > max_chunk_size:
    # 仅当单个章节内容超长时才按段落切分
    flush_chunk()
```

### 3️⃣ 测试集智能过滤器

#### 新增过滤规则

##### 规则 1：乱码检测 (`is_garbled_text()`)

**功能**：识别随机大写字母序列（如 "SRAFY"、"QWERTY"）

```python
def is_garbled_text(text: str, min_length: int = 4) -> bool:
    """
    检测文本是否包含乱码（随机大写英文字符串）
    
    规则：
    - 匹配连续 min_length 个及以上大写字母
    - 排除常见合法模式（缩写、型号、数学符号）
    - 使用白名单 + 黑名单双重校验
    
    示例:
        >>> is_garbled_text("答案是 SRAFY")  # True（随机大写）
        >>> is_garbled_text("使用 ResNet-50")  # False（已知型号）
    """
```

**白名单模式**（不视为乱码）：
- 型号格式：`ResNet-50`, `BERT-base`
- 大O表示法：`O(n^2)`
- 短缩写：`AI`, `ML`, `GPU`, `CPU`
- 驼峰命名：`TensorFlow`, `PyTorch`

##### 规则 2：重复内容检测 (`is_duplicate_content()`)

**功能**：检测答案是否仅为问题公式的简单重复

```python
def is_duplicate_content(question: str, answer_point: str) -> bool:
    """
    检测答案是否仅为问题中公式/关键词的简单重复
    
    判定流程：
    1. 标准化处理（去除空格、标点、换行）
    2. 提取问题中的非中文序列（公式/术语/代码）
    3. 检查答案是否完全由这些提取内容组成
    4. 排除有实质内容的回答（含"含义"、"表示"、"是指"等）
    
    示例:
        >>> is_duplicate_content("d*d的含义", "d*d")  # True
        >>> is_duplicate_content("d*d的含义", "d*d是维度乘积")  # False
    """
```

##### 规则 3：文档支撑验证升级 (`check_document_support()`)

**新标准**：连续匹配字符长度 ≥ **5字符**（替代旧的前10字符匹配规则）
