## 🆕 新增功能

### 1. Prompt 优化装置

**文件**: `core/prompt_optimizer.py` (新增)

**功能描述**:
- 解决用户输入过于简短导致模型输出不充分的问题
- 自动对 prompt 进行规范化处理和信息扩充
- 利用现有检索架构获取相关资料
- 生成内容充分、结构完整的优化后 prompt

**核心特性**:
- ✅ 智能问题类型分类（what/how/why/compare/application）
- ✅ 自动主题提取
- ✅ 相关资料检索（C++ 粗排 + ONNX 精排）
- ✅ 结构化 prompt 生成
- ✅ 不依赖 LLM 和 CUDA 模块
- ✅ 与现有架构无缝集成

**示例**:
| 输入 | 优化后效果 |
|------|-----------|
| "什么是PPO" | 生成 2398 字符的完整 prompt，包含概念、原理、应用 |
| "如何实现DDP" | 生成 2312 字符的实现指南，包含步骤和示例 |

**新增接口**:
```python
# core/engine.py
async def optimize_prompt(self, original_prompt: str, top_k: int = 10) -> Dict[str, Any]
```

### 2. 环境变量管理系统

**文件**: `core/env_manager.py` (新增)

**功能描述**:
- 统一管理环境变量，防止敏感信息硬编码
- 支持从 .env 文件加载配置
- 提供类型转换和默认值机制
- 确保敏感信息仅在本地环境使用

**核心类**:
```python
class EnvironmentManager:
    def get(self, key: str, default=None) -> str
    def get_bool(self, key: str, default=False) -> bool
    def get_int(self, key: str, default=0) -> int
    def get_float(self, key: str, default=0.0) -> float
```

**配置项**:
| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| OLLAMA_URL | http://localhost:11434 | Ollama API 地址 |
| OLLAMA_MODEL | qwen2.5:7b | 使用的模型 |
| CACHE_DIR | ./cache | 缓存目录 |
| EMBEDDING_MODEL_PATH | models/bge-small-zh-onnx/model.onnx | Embedding 模型路径 |
| RERANKER_MODEL_PATH | models/crossencoder-fp32/model.onnx | Reranker 模型路径 |
| EVALUATION_THRESHOLD | 0.55 | 语义评估阈值 |
