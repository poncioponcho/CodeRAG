
# 2026-04-21｜CodeRAG 本地知识库搭建与全链路踩坑总结

> 目标：放弃不稳定的 SQLite 玩具项目，搭建基于本地向量检索的 CodeRAG 知识库问答系统，并完成全链路跑通。
>
> 最终架构：本地 Markdown → ChromaDB 向量存储 → 向量检索(k=3) → 手动 RAG Prompt → Ollama 原生 HTTP 生成
>
> LLM：qwen2.5:7b (Ollama) | Embedding：all-MiniLM-L6-v2 | 向量库：ChromaDB

---

## 一、项目选型决策

### 1.1 放弃旧项目的原因

| 维度 | SQLite 玩具项目 | CodeRAG Agent |
|------|----------------|---------------|
| 技术深度 | 单表 SQL | 文档切分、Embedding、向量检索、RAG Pipeline |
| 环境稳定性 | Ollama tool calling 反复 503 | 向量库全本地离线；LLM 走稳定接口 |
| 面试价值 | 无 | 覆盖 RAG + Agent + 本地部署三大关键词 |
| 实用价值 | 无 | 直接查询学习笔记/代码，复习面试可用 |

### 1.2 架构选择：纯本地 RAG（非 Agent）

原计划使用 `create_react_agent` + ReAct 架构，但 Ollama 0.21 + qwen2.5:7b 的 tool calling 存在兼容 bug，最终降级为**手动 RAG Pipeline**：
- 向量检索 + 手动 Prompt 拼接 + 直接调用 LLM
- 零 tool calling 依赖，稳定性 100%

## 四、代码迭代过程

### Step 0：初版 04_coderag_agent.py（Agent 架构）

**设计意图**：使用 `create_react_agent` + `create_retriever_tool` 实现 ReAct Agent。

**遇到的问题与修复**：

| 轮次 | 问题 | 根因 | 修复方案 |
|------|------|------|---------|
| 1 | `ModuleNotFoundError: langchain.tools.retriever` | LangChain 1.x 模块重组，`create_retriever_tool` 迁移 | 改用 `@tool` 手动封装 retriever |
| 2 | `ModuleNotFoundError: langchain.text_splitter` | `langchain.text_splitter` 拆分为独立包 `langchain-text-splitters` | 修正导入路径 |
| 3 | 切分为 0 个 chunk → Chroma 报错 | `DirectoryLoader` 隐式兼容 bug，读到文件数但没读到内容 | 弃用 DirectoryLoader，改用 `os.listdir` + `open` 手动读取 + `Document` 显式封装 |
| 4 | Ollama 503 / 404 | OpenRouter 免费模型下架；Ollama /v1 兼容层不稳定 | 最终弃用外部 API 和 /v1 兼容层 |

### Step 1：终版 05_local_rag.py（纯本地 RAG）

**核心改动**：

| 组件 | 初版 | 终版 | 原因 |
|------|------|------|------|
| LLM 接口 | `ChatOpenAI(base_url=".../v1")` | `requests.post("/api/generate")` 原生 HTTP | /v1 兼容层对中文长 prompt 不稳定 |
| LLM 封装 | `ChatOllama` (LangChain) | 原生 `requests` 调用 | `langchain-ollama` 封装层有 bug |
| Agent 架构 | `create_react_agent` + tool calling | 手动 RAG Pipeline | qwen2.5:7b tool calling 兼容性差 |
| 向量库加载 | 每次重建 | 检测存在则直接加载 | 避免重复 Embedding |

**最终稳定代码核心逻辑**：

```python
# 1. 手动读取文档 + 切分
docs = []
for filename in os.listdir("docs"):
    if filename.endswith(".md"):
        with open(f"docs/{filename}", "r", encoding="utf-8") as f:
            docs.append(Document(page_content=f.read(), metadata={"source": filename}))

splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(docs)

# 2. 向量库（存在则加载，否则重建）
if os.path.exists("./chroma_db") and os.listdir("./chroma_db"):
    vectorstore = Chroma(persist_directory="./chroma_db", embedding_function=embedding)
else:
    vectorstore = Chroma.from_documents(chunks, embedding, persist_directory="./chroma_db")

# 3. 检索 + RAG Prompt 拼接
docs = retriever.invoke(query)
context = "\n\n".join([f"[{d.metadata['source']}] {d.page_content}" for d in docs])

prompt = f"""你是技术面试助手。基于以下参考资料用中文回答问题。
如果参考资料中没有相关信息，请明确说明"资料中未提及"。
参考资料：{context}
历史对话：{history_str}
当前问题：{query}
请回答："""

# 4. 原生 HTTP 调用 Ollama（最稳定路径）
def ollama_generate(prompt: str) -> str:
    resp = requests.post(
        "http://localhost:11434/api/generate",
        json={"model": "qwen2.5:7b", "prompt": prompt, "stream": False, "options": {"temperature": 0.1}}
    )
    resp.raise_for_status()
    return resp.json()["response"]
```

## 五、全量踩坑汇总

### 5.1 Ollama 服务端问题

| 报错现象 | 根因 | 解法 |
|---------|------|------|
| `could not connect to ollama server` | 服务未启动 | `OLLAMA_KEEP_ALIVE=-1 ollama serve` 常驻内存 |
| `bind: address already in use` | 旧进程未死 | `pkill ollama` → `lsof -i :11434` 确认 → 重启动 |
| 503（ChatOpenAI + /v1） | OpenAI 兼容层对中文长 prompt / tool calling 不稳定 | **放弃 /v1**，改用原生 HTTP `/api/generate` |
| 503（ChatOllama + bind_tools） | qwen2.5:7b function calling 有 bug | **放弃 tool calling**，改用手动 RAG |
| 模型下载完就 503 | 模型未加载到显存 | `ollama run qwen2.5:7b` 预热后再跑 Python |
| 跑通后过会儿又 503 | 默认 5 分钟无请求自动卸载 | `OLLAMA_KEEP_ALIVE=-1` 永久驻留 |

### 5.2 LangChain 1.x 模块重组

| 报错 | 根因 | 解法 |
|------|------|------|
| `No module named 'langchain.text_splitter'` | 拆分为独立包 `langchain-text-splitters` | `from langchain_text_splitters import ...` |
| `No module named 'langchain.tools.retriever'` | `create_retriever_tool` 被移除 | 手动 `@tool` 封装，零版本依赖 |
| `create_react_agent is deprecated` | langgraph 版本迭代 | 黄色警告不影响运行；或 `warnings.filterwarnings("ignore", ...)` |

### 5.3 外部 API 问题（最终全部弃用）

| 报错 | 根因 | 解法 |
|------|------|------|
| 404 `google/gemini-2.0-flash-exp:free` | OpenRouter 下架免费端点 | 换其他免费模型 |
| 429 `account suspended` | 账户余额耗尽 | **彻底放弃外部 API**，全切本地 Ollama |

### 5.4 RAG Pipeline 数据问题

| 报错 | 根因 | 解法 |
|------|------|------|
| `Expected Embeddings to be non-empty list, got []` | DirectoryLoader 兼容 bug，0 个 chunk | `os.listdir` + `open` 手动读取 + `Document` 封装 |
| `chunk_size=500` 但只切出 3 个 chunk | 示例文档太短 | 正常结果，换真实长文档后自然增加 |

### 5.5 IDE / 环境噪音

| 现象 | 根因 | 解法 |
|------|------|------|
| Pylance 波浪线：`Type "str" is not assignable to "SecretStr"` | Pylance 严格类型检查与 LangChain 类型注解不匹配 | 加 `# type: ignore`，或无视 |
| `Warning: unauthenticated requests to HF Hub` | HuggingFaceEmbeddings 未登录 | 非阻塞警告，可无视 |

## 八、前置排查 Checklist

1. Ollama 是否常驻？`OLLAMA_KEEP_ALIVE=-1 ollama serve`
2. 是否走了 /v1 兼容层？纯文本用 `/api/generate`，不用 `ChatOpenAI(base_url=".../v1")`
3. 是否需要 tool calling？本地 7B 模型 tool calling 不稳定，直接手写 RAG Prompt
4. LangChain 导入是否新路径？不确定就 `pip show langchain-text-splitters` 确认

6. API 是否可用？免费层随时下架/限流，默认走本地
