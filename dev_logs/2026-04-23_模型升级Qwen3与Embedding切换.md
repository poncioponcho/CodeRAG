## 2. 更新目的与背景说明

### 2.1 更新目的
为提升CodeRAG系统的问答质量和检索性能，将核心AI模型从Qwen 2.5 7B升级至Qwen 3 7B，并同步优化相关组件配置。

### 2.2 背景说明
- 原系统使用Qwen 2.5 7B模型作为核心推理引擎
- Qwen 3系列模型在推理能力、上下文理解和响应质量方面有显著提升
- 需要确保升级后系统兼容性和稳定性

---

## 3. 具体更新内容

### 3.1 模型版本变更

| 组件 | 原版本 | 新版本 | 变更说明 |
|------|--------|--------|----------|
| 核心LLM模型 | qwen2.5:7b | qwen3 | 通过Ollama进行模型替换 |
| Embedding模型 | all-MiniLM-L6-v2 | BAAI/bge-small-zh | 提升文本向量化质量 |
| 向量索引 | FAISS (384维) | FAISS (768维) | 重建索引适配新Embedding |

### 3.2 新增功能模块

| 模块名称 | 功能描述 | 状态 |
|----------|----------|------|
| `question_classifier.py` | 问题类型分类器，区分抽象问题与具体技术问题 | ✅ 已完成 |
| `hyde_module.py` | HyDE生成器，为抽象问题生成假设性答案 | ✅ 已完成 |
| `rebuild_vectorstore.py` | 向量库重建脚本 | ✅ 已完成 |

### 3.3 修改的核心文件

| 文件路径 | 修改内容 |
|----------|----------|
| `app.py` | 更新模型引用为qwen3，集成HyDE功能，替换embedding模型 |
| `retrieval_core.py` | 新增HyDERetriever类，修改RerankRetriever支持HyDE |
| `retrieval_plugins.py` | ContextDenoisePlugin适配新embedding模型 |
| `evaluate_batch.py` | 实现三种消融试验评估逻辑 |

## 6. 版本变更前后差异说明

### 6.1 性能指标对比

| 指标 | 变更前 (Qwen 2.5) | 变更后 (Qwen 3) | 变化 |
|------|-------------------|------------------|------|
| 检索覆盖率 | ~45% | 53.8% | +8.8% |
| 回答准确性 | ~3.5 | 3.8 | +8.6% |
| 相关性评分 | ~4.0 | 4.4 | +10% |
| 检索延迟 | ~800ms | 628ms | -21.5% |
| 生成延迟 | ~18s | 15.5s | -13.9% |

### 6.2 功能差异

| 功能 | 变更前 | 变更后 |
|------|--------|--------|
| HyDE支持 | ❌ 不支持 | ✅ 支持（仅对抽象问题启用） |
| 问题分类 | ❌ 无 | ✅ 规则匹配+LLM分类 |
| 去噪处理 | ✅ 基础 | ✅ 增强（bge-small-zh） |
| 消融试验 | ❌ 无 | ✅ 支持三种试验 |

### 6.3 架构差异

```
变更前架构:
┌─────────────────────────────────────────────────┐
│  Query → Hybrid检索 → Rerank → Qwen 2.5 → Answer│
└─────────────────────────────────────────────────┘

变更后架构:
┌─────────────────────────────────────────────────────────────┐
│  Query → 问题分类 → HyDE(可选) → Hybrid检索 → 去噪 → Rerank │
│                                              ↓              │
│                                    Qwen 3 → Answer          │
└─────────────────────────────────────────────────────────────┘
```

## 8. 备注与注意事项

1. **模型文件位置**：Qwen 3模型存储于Ollama默认目录，不占用项目存储空间

3. **Ollama清理**：如需释放存储空间，建议执行`ollama rm qwen2.5`删除旧模型
