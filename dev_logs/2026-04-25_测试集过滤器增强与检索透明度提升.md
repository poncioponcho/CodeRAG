## 2. 更新目的与背景说明

### 2.1 更新目的

本次更新包含两个独立但相关的改进需求：

**需求一：测试集过滤器增强**
- 实现智能化的低质量样本识别与过滤机制
- 提升测试集数据质量，确保评估结果的可靠性
- 增强过滤过程的可观测性和可追溯性

**需求二：检索过程透明度提升**
- 在对话界面中展示完整的检索链路信息
- 提升系统的可解释性和可信度
- 支持开发者调试和性能优化分析

### 2.2 背景说明

#### 问题发现（过滤器）

当前 `filter_test_set.py` 存在以下不足：

| 问题类型 | 具体表现 | 影响 |
|----------|----------|------|
| **无法识别乱码** | 随机大写字母串（如 "SRAFY"）被当作正常答案 | 污染评估数据 |
| **无法检测重复** | 答案仅为问题公式的简单重复（如 "d*d"→"d*d"） | 虚假高质量样本 |
| **匹配标准粗糙** | 前10字符匹配规则过于宽松 | 边界 case 误判 |
| **缺乏分类统计** | 无法区分不同类型的丢弃原因 | 问题定位困难 |

#### 问题发现（检索面板）

当前对话界面存在以下问题：

| 问题类型 | 具体表现 | 影响 |
|----------|----------|------|
| **过程不透明** | 用户无法看到检索的中间结果 | 信任度降低 |
| **调试困难** | 开发者难以定位检索问题 | 效率低下 |
| **数据不可追溯** | 无法复现或分析某次检索的具体行为 | 经验积累受阻 |

#### 业务影响

- 测试集质量直接影响评估结果的准确性
- 检索透明度影响用户体验和系统可信度
- 缺乏可观测性导致问题排查成本高

```python
def is_garbled_text(text: str, min_length: int = 4) -> bool:
    """
    检测文本是否包含乱码（随机大写英文字符串）
    
    规则：
    - 匹配连续 min_length 个及以上大写字母
    - 排除常见合法模式（缩写、型号、数学符号）
    - 使用白名单 + 黑名单双重校验
    
    示例:
        >>> is_garbled_text("答案是 SRAFY")  # True（随机大写）
        >>> is_garbled_text("使用 ResNet-50")  # False（已知型号）
    """
```

**正则模式**：
```python
# 匹配目标：连续4+大写字母
pattern = rf'[A-Z]{{{min_length},}}'

# 白名单（不视为乱码）
legitimate_patterns = [
    r'^[A-Z][a-z]*-[0-9]+$',       # 型号格式: ResNet-50
    r'O\([^)]+\)$',                 # 大O表示法
    r'\b[A-Z]{2,3}\b(?![A-Z])',     # 短缩写 (AI, ML, GPU)
    r'[A-Z][a-z]+[A-Z][a-z]*',      # 驼峰命名 (TensorFlow)
    ...
]
```

##### 2️⃣ 重复内容检测机制

```python
def is_duplicate_content(question: str, answer_point: str) -> bool:
    """
    检测答案是否仅为问题中公式/关键词的简单重复
    
    判定流程：
    1. 标准化处理（去除空格、标点、换行）
    2. 提取问题中的非中文序列（公式/术语/代码）
    3. 检查答案是否完全由这些提取内容组成
    4. 排除有实质内容的回答（含"含义"、"表示"、"是指"等）
    
    示例:
        >>> is_duplicate_content("d*d的含义", "d*d")  # True
        >>> is_duplicate_content("d*d的含义", "d*d是维度乘积")  # False
    """
```

**关键算法**：
```python
# 提取非中文序列
formula_pattern = r'[a-zA-Z0-9_\-\+\*\/\^\(\)\[\]\{\}\=\&\|\!\<\>\,\.\:\;]+'
formulas_in_question = set(re.findall(formula_pattern, q_normalized))

# 检查答案是否完全由这些公式组成
for formula in formulas_in_question:
    if formula and len(formula) >= 2:
        if a_normalized == formula or a_normalized in formula:
            return True
```

##### 3️⃣ 文档支撑验证升级

```python
def check_document_support(point: str, document_text: str, min_match_length: int = 5) -> bool:
    """
    检查答案要点是否在文档中有足够的支撑
    
    新标准（替代旧的前10字符规则）：
    - 连续匹配字符长度 ≥ min_match_length（默认5）
    - 多层次验证策略：
      1. 直接精确匹配
      2. 子串匹配（≥5字符）
      3. 关键词组合匹配
      4. 语义匹配兜底（阈值0.75）
    
    特殊处理：
    - 短字符串（<5字符）检查周围上下文
    - 排除空格和特殊符号干扰
    - 停用词过滤
    """
```

**匹配策略层次**：
```python
# 策略1: 直接精确匹配
if point_lower in doc_clean and len(point_clean) >= min_match_length:
    return True

# 策略2: 子串匹配（≥min_match_length）
words = re.split(r'[，。；：！？、\s]+', point_clean)
for word in words:
    if len(word) >= min_match_length and word.lower() in doc_clean:
        return True

# 策略3: 关键词组合匹配
key_parts = [w for w in words if w not in stopwords and len(w) >= 2]
if sum(len(p) for p in matched_parts) >= threshold:
    return True

# 策略4: 语义匹配兜底
if semantic_match(point, document_text, threshold=0.75):
    return True
```

## 8. 发现问题与解决方案

### 8.1 技术挑战与应对

#### 挑战 1：乱码检测的误判控制

**问题**：如何区分真正的乱码和合法的大写字母组合？

**解决方案**：
- 采用**白名单优先策略**：先检查是否符合已知合法模式（型号、缩写、数学符号）
- 使用**负向预查**：`(?![A-Z])` 确保短缩写后面不再跟大写字母
- **分层验证**：整体文本匹配 → 子串匹配 → 最后判定

**效果**：
- ✅ 正确识别 "SRAFY"、"QWERTY"、"XYZABC" 为乱码
- ✅ 正确放行 "ResNet-50"、"AI"、"O(n^2)"、"TensorFlow"

---

#### 挑战 2：重复检测的语义理解

**问题**：如何判断"重复"而非"相关"?

**解决方案**：
- **严格定义**：仅当答案完全由问题中的非中文序列组成时才判定为重复
- **排除策略**：检查是否有解释性词汇（"含义"、"表示"、"是指"、"即"、"是"）
- **标准化处理**：去除空格、标点后再比对，避免格式差异干扰

**边界案例**：
```python
# 应该判为重复
is_duplicate("d*d的含义", "d*d")  # → True

# 不应该判为重复（有额外信息）
is_duplicate("d*d的含义", "d*d是维度乘积")  # → False
is_duplicate("ResNet-50", "ResNet-50是一个经典模型")  # → False
```

---

#### 挑战 3：检索数据获取的兼容性

**问题**：如何在现有架构上添加详细信息返回而不破坏向后兼容？

**解决方案**：
- 使用**可选参数** `return_details=False`（默认值保持原有行为）
- 返回值类型判断：`isinstance(result, dict)` 区分新旧格式
- **渐进式迁移**：HyDE 模式自动降级为简单显示

**代码示例**：
```python
# 旧调用方式（仍然有效）
docs = retriever.invoke(query)

# 新调用方式（获取详细信息）
details = retriever.invoke(query, return_details=True)
# details = {
#     "results": [...],
#     "vec_results": [...],
#     "rerank_scores": [...]
# }
```
