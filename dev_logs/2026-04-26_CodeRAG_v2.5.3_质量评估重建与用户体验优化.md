### 2. 语义评估器 (Semantic Evaluator)

**文件**: `evaluation/semantic_evaluator.py` (新增)

**功能描述**:
- 使用 `paraphrase-multilingual-MiniLM-L12-v2` 模型计算语义相似度
- 替代有严重缺陷的字符串子串匹配方法
- 支持批量评估，同时输出语义/字符串两种覆盖率对比

**核心类**:
```python
class SemanticEvaluator:
    def __init__(self, model_name='paraphrase-multilingual-MiniLM-L12-v2',
                 threshold=0.55, device='cpu')

    def compute_coverage(self, answer, expected_points) -> Tuple[float, List[Dict]]
    def evaluate_single(self, query, answer, expected_points) -> EvaluationResult
    def evaluate_batch(self, results, verbose=True) -> Dict
    def validate_with_examples(self) -> Dict
```

**核心优势**:
- ✅ 能识别同义词替换（如"缺乏" vs "缺少"）
- ✅ 能识别语序变换（如主语宾语互换）
- ✅ 能识别补充说明和解释性内容
- ✅ 更接近人类对"好回答"的判断

**验证结果** (100% 通过率):

| 语序变换 | 100% | 0% | +100pp |

**文件**: `core/reranker.py`

**问题**: CoreML 初始化成功但推理时崩溃 (error code: -1)，导致静默 fallback

**解决方案**:
```python

providers = ['CPUExecutionProvider']
self.session = ort.InferenceSession(model_path, providers=providers)
```

**效果**:
- ✅ 消除 error code -1 崩溃
- ✅ 延迟稳定在 8.10ms (P95: 11.49ms)
- ✅ 结果可重现，相同输入始终产生相同输出

## 🐛 Bug 修复

### 1. 流式输出 async generator 错误

**问题**: `'async_generator' object is not iterable`

**根因**: `stream_wrapper()` 是 async generator，不能直接用 `list()` 转换

**修复**:
```python
# 修复前（错误）
async def stream_wrapper():
    async for chunk in engine.stream_query(...):
        yield chunk
chunks = run_async(lambda: list(stream_wrapper()))  # TypeError

# 修复后（正确）
async def collect_chunks():
    chunks_list = []
    async for chunk in engine.stream_query(...):
        chunks_list.append(chunk)
    return chunks_list
chunks = run_async(collect_chunks)
```

## 🔬 核心发现与教训

### 1. 评估方法选择至关重要

**发现**:
- 字符串匹配严重低估系统真实能力（-50pp）
- 语义评估更接近真实质量

**教训**:
- 不要基于有缺陷的指标做优化决策
- 先建立可靠的评估体系，再做参数调优

### 2. 稳定性优先于极致性能

**案例**: Reranker CoreML 崩溃
- 追求 2-4ms (CoreML) 导致不确定性和崩溃
- 稳定的 8ms (CPU) 远优于不稳定的 2-4ms

**教训**: 先保证可用性，再追求性能优化

### 3. 基线是宝贵的资产

**案例**: Prompt/Temperature 调优
- 没有可靠基线的调优是盲人摸象
- 任何偏离基线的改动都需要充分验证

**教训**: 回归基线是消除变量、确保可复现性的最佳策略
