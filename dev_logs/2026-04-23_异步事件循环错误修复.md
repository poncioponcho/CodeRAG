## 2. 问题描述

### 2.1 错误信息
```
RuntimeError: There is no current event loop in thread 'ScriptRunner.scriptThread'.
```

### 2.2 错误堆栈
```
1. app.py 第194行：base_retriever = HyDERetriever(hybrid_retriever, hyde_generator)
2. retrieval_core.py 第108行：self.parallel_processor = get_parallel_processor(max_workers=max_workers)
3. parallel_processor.py 第139行：_parallel_processor = ParallelProcessor(max_workers=max_workers)
4. parallel_processor.py 第20行：self.loop = asyncio.get_event_loop()
5. asyncio/events.py 第716行：raise RuntimeError('There is no current event loop in thread %r.' % threading.current_thread().name)
```

### 2.3 影响范围
- Streamlit 前端启动失败
- 应用无法正常运行

---

## 3. 根本原因分析

### 3.1 Python 3.10+ 行为变化
在 Python 3.10 之前，`asyncio.get_event_loop()` 在没有事件循环的线程中会自动创建一个新的事件循环。但从 Python 3.10 开始（特别是 3.13），这种行为被移除了，在非主线程中调用会抛出 `RuntimeError`。

### 3.2 Streamlit 线程模型
Streamlit 的 `ScriptRunner.scriptThread` 是一个工作线程，不是主线程，因此没有默认的事件循环。

### 3.3 代码问题
`parallel_processor.py` 在 `__init__` 中直接调用 `asyncio.get_event_loop()`，导致在 Streamlit 的工作线程中初始化时失败。

---

## 4. 修复方案

### 4.1 方案选择
选择**方案2：移除事件循环依赖**

**原因**：
- 当前的 `ParallelProcessor` 实际上使用的是 `ThreadPoolExecutor`（多线程），而不是真正的异步IO
- 移除不必要的 asyncio 依赖可以简化代码
- 提高线程安全性
- 兼容 Python 3.8+ 所有版本

### 4.2 代码修改

**修改文件**: `parallel_processor.py`

**修改内容**:
1. 移除 `asyncio` 导入和事件循环相关代码
2. 将 `process_query` 方法改为同步方法（移除 `async` 关键字）
3. 将 `process_batch` 方法改为同步方法（移除 `async` 关键字）
4. 移除 `AsyncProcessor` 类（暂时不需要）

**修改文件**: `retrieval_core.py`

**修改内容**:
1. 移除 `import asyncio` 语句
2. 修改 `invoke` 方法中对 `process_query` 的调用方式，直接调用同步方法
